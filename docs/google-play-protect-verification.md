# Google Play Protect and Android Developer Verification

This app is distributed from the DlaFlow panel as a signed APK. The current production strategy is to keep that distribution path and register the app with Google as a verified Android developer/package, without publishing the app to Google Play Store yet.

## Decision

- Primary distribution: DlaFlow panel plugin, HTTPS APK download from the VPS release directory.
- Package name: `pl.dlaflow.mobile`.
- Signing standard: one stable release signing key from GitHub Secrets. Do not install debug builds on customer phones.
- Google path: Android Developer Verification package registration first. Google Play Store publication remains a later optional channel.

## Why this path

Google supports developer verification and package registration for apps distributed outside Google Play. This gives users a safer install path without forcing the app into the Play Store review and listing process immediately.

Official references:

- Android Developer Verification: https://developer.android.com/developer-verification
- Android Developer Verification FAQ: https://developer.android.com/developer-verification/guides/faq
- Google Play app verification appeal: https://support.google.com/googleplay/android-developer/answer/2992033
- Android app signing: https://developer.android.com/studio/publish/app-signing

## Registration data

Use a signed production APK generated by the `mobile-v<versionName>` release workflow. Do not use a debug APK.

Generate the registration kit:

```powershell
.\scripts\export-google-play-protect-kit.ps1 `
  -ApkPath ".\mobile-release\dlaflow-mobile-assistant-0.3.1-production.apk" `
  -OutputDir ".\mobile-release\google-play-protect"
```

The generated files contain only public metadata:

- package name,
- version name and code,
- APK SHA-256,
- signing certificate fingerprints,
- local path of the APK used for proof,
- checklist for the Google console.

The script never reads or exports the private signing key.

## Google console steps

1. Open Android Developer Console or Play Console with the DlaFlow organization account.
2. Complete developer identity verification if Google asks for it.
3. Register package name `pl.dlaflow.mobile`.
4. Upload or provide the signed production APK requested by Google to prove ownership of the package/signing key.
5. Confirm the package status changes to a registered/verified state.
6. Keep the generated registration kit in local operator storage only if it contains local paths. Do not commit APK files.
7. After registration, install the current production APK from the DlaFlow panel on a physical Android phone and confirm Play Protect does not block it.

## Store publication is separate

Publishing in Google Play Store is not required for this phase. If we later add it, use a separate Play Store launch checklist:

- app listing,
- privacy policy and Data safety form,
- permissions declaration,
- screenshots,
- review track,
- Play App Signing decision,
- Play Integrity API decision.

## If Play Protect still blocks the APK

If a verified, signed production APK is still blocked or flagged, use Google's app verification appeal flow. Attach the package name, APK SHA-256, signing certificate SHA-256, version name, version code, and a short business explanation that this is the DlaFlow Mobile Assistant distributed to authenticated DlaFlow panel users.
